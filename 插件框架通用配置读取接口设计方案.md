# 插件框架通用配置读取接口设计方案

## 1. 需求背景

为multimodalinput插件框架扩展**通用的配置读取能力**，使任何插件都能够通过key读取Settings数据库中的value，并监听配置变更。

### 1.1 核心目标

1. 扩展IPluginContext接口，提供通过key读取value的能力
2. 在InputPlugin中实现这些接口，对接现有的SettingManager和SettingDataShare
3. **框架只负责读取和监听，不关心value的具体格式**
4. value的解析（int、string、json等）由插件自行实现
5. **提供DataShare就绪状态感知能力，避免未就绪时的阻塞问题**

### 1.2 设计原则

- **极简通用**：只提供key→value的读取，不关心value格式
- **插件自主**：value的解析完全由插件决定（可以是int、string、json等）
- **底层能力**：框架提供最基础的数据库访问能力
- **高效安全**：利用SettingManager的缓存和白名单机制
- **防止阻塞**：提供DataShare就绪状态查询，避免未就绪时的读取和监听操作

## 2. 系统架构

### 2.1 整体架构

```
任意插件 (鼠标、键盘、触摸板等)
    ↓ 调用GetSettingValue(key)
IPluginContext接口 (本次扩展)
    ↓ 实现
InputPlugin类 (本次实现)
    ↓ 委托给
SettingManager (现有配置系统)
    ↓ 访问
Settings数据库
    ↓ 返回
原始value (string/int/json等)
    ↓ 插件自行解析
插件业务逻辑
```

### 2.2 为什么返回字符串？

1. **通用性**：Settings数据库中的value可以是int、bool、string、json等多种类型
2. **灵活性**：插件可以根据自己的需求解析成任何类型
3. **简洁性**：框架不需要提供多个重载方法（GetIntValue、GetBoolValue、GetStringValue等）
4. **扩展性**：未来支持新的数据类型无需修改框架接口

## 3. 接口设计

### 3.1 GetSettingValue - 读取配置值

```cpp
virtual bool GetSettingValue(
    const std::string& uri,
    const std::string& key,
    std::string& value) = 0;
```

**功能**：通过URI和key读取Settings数据库中的value（以字符串形式返回）

**参数**：
- `uri`: DataShare URI，如"datashare:///com.ohos.settingsdata/entry/settingsdata/USER_SETTINGSDATA_SECURE_100"
- `key`: 配置键，如"settings.input.mouse.scrollRows"、"settings.input.custom.config"等
- `value`: 输出参数，返回配置值的字符串表示

**返回值**：
- `true`: 读取成功
- `false`: 读取失败（URI无效、key不存在等）

**设计优势**：
- **更通用**：支持访问不同的DataShare数据源
- **更灵活**：插件可以指定具体的用户数据或系统数据
- **解耦用户ID**：框架不需要自动获取用户ID，由插件决定访问哪个用户的数据

**使用示例**：
```cpp
// 构造URI（插件可以根据需要访问不同用户的数据）
int32_t userId = context->GetCurrentAccountId();
std::string uri = "datashare:///com.ohos.settingsdata/entry/settingsdata/USER_SETTINGSDATA_SECURE_"
                  + std::to_string(userId);

// 读取整型配置
std::string valueStr;
if (context->GetSettingValue(uri, "settings.input.mouse.scrollRows", valueStr)) {
    int scrollRows = std::stoi(valueStr);  // 插件自行解析
}

// 读取JSON配置
std::string jsonStr;
if (context->GetSettingValue(uri, "settings.input.custom.config", jsonStr)) {
    auto config = nlohmann::json::parse(jsonStr);  // 插件使用json库解析
}

// 读取布尔配置
std::string boolStr;
if (context->GetSettingValue(uri, "settings.input.mouse.enabled", boolStr)) {
    bool enabled = (boolStr == "true" || boolStr == "1");
}
```

### 3.2 RegisterSettingObserver - 注册配置监听

```cpp
virtual sptr<SettingObserver> RegisterSettingObserver(
    const std::string& uri,
    const std::string& key,
    SettingObserver::UpdateFunc callback) = 0;
```

**功能**：注册配置变更监听器，当指定URI和key的value变更时触发回调

**参数**：
- `uri`: DataShare URI
- `key`: 要监听的配置键
- `callback`: 回调函数，签名为 `void(const std::string& key)`

**返回值**：
- 成功：返回观察者指针（用于后续注销）
- 失败：返回nullptr

**使用示例**：
```cpp
int32_t userId = context->GetCurrentAccountId();
std::string uri = "datashare:///com.ohos.settingsdata/entry/settingsdata/USER_SETTINGSDATA_SECURE_"
                  + std::to_string(userId);

auto callback = [this, uri](const std::string& key) {
    std::string newValue;
    if (context_->GetSettingValue(uri, key, newValue)) {
        int newScrollRows = std::stoi(newValue);
        OnConfigChanged(newScrollRows);
    }
};

observer_ = context_->RegisterSettingObserver(
    uri,
    "settings.input.mouse.scrollRows",
    callback);
```

### 3.3 UnregisterSettingObserver - 注销配置监听

```cpp
virtual bool UnregisterSettingObserver(
    const std::string& uri,
    sptr<SettingObserver> observer) = 0;
```

**功能**：注销之前注册的配置监听器

**参数**：
- `uri`: DataShare URI（与注册时使用的URI相同）
- `observer`: RegisterSettingObserver返回的观察者指针

**返回值**：
- `true`: 注销成功
- `false`: 注销失败

### 3.4 IsDataShareReady - 查询DataShare就绪状态

```cpp
virtual bool IsDataShareReady() = 0;
```

**功能**：查询Settings DataShare是否已就绪，可以安全进行读取和监听操作

**返回值**：
- `true`: DataShare已就绪，可以安全调用GetSettingValue和RegisterSettingObserver
- `false`: DataShare未就绪，调用读取和监听接口可能导致阻塞

**使用示例**：
```cpp
class MyPlugin : public IInputPlugin {
public:
    void Init(std::shared_ptr<IPluginContext> context) {
        context_ = context;

        // 检查DataShare是否就绪
        if (!context_->IsDataShareReady()) {
            MMI_HILOGW("DataShare not ready, defer configuration loading");
            // 可以选择延迟加载或使用默认配置
            return;
        }

        // 构造URI
        int32_t userId = context_->GetCurrentAccountId();
        uri_ = "datashare:///com.ohos.settingsdata/entry/settingsdata/USER_SETTINGSDATA_SECURE_"
               + std::to_string(userId);

        // DataShare已就绪，安全读取配置
        std::string valueStr;
        if (context_->GetSettingValue(uri_, "settings.input.mouse.scrollRows", valueStr)) {
            int scrollRows = std::stoi(valueStr);
            MMI_HILOGI("Mouse scroll rows: %{public}d", scrollRows);
        }
    }

private:
    std::string uri_;
};
```

**重要性**：
- 避免在DataShare未就绪时调用读取和监听接口导致的阻塞
- 插件可以根据就绪状态决定是使用默认配置还是延迟加载
- 提高系统启动性能和稳定性

bool InputPlugin::GetSettingValue(const std::string& uri,
                                  const std::string& key,
                                  std::string& value)
{
    // Validate parameters
    if (uri.empty() || key.empty()) {
        MMI_HILOGE("Invalid parameters: uri or key is empty");
        return false;
    }

    // Extract userId from URI
    // URI format: "datashare:///com.ohos.settingsdata/entry/settingsdata/USER_SETTINGSDATA_SECURE_100"
    size_t pos = uri.rfind('_');
    if (pos == std::string::npos) {
        MMI_HILOGE("Invalid URI format: %{public}s", uri.c_str());
        return false;
    }

    int32_t userId = 0;
    try {
        userId = std::stoi(uri.substr(pos + 1));
    } catch (const std::exception& e) {
        MMI_HILOGE("Failed to parse userId from URI: %{public}s", e.what());
        return false;
    }

    // Delegate to SettingManager
    auto settingMgr = INPUT_SETTING_MANAGER;
    if (settingMgr == nullptr) {
        MMI_HILOGE("Setting manager is null");
        return false;
    }

    // Parse key: "settings.input.mouse.scrollRows" -> settingKey + field
    size_t lastDot = key.rfind('.');
    if (lastDot == std::string::npos) {
        MMI_HILOGE("Invalid key format: %{public}s", key.c_str());
        return false;
    }

    std::string settingKey = key.substr(0, lastDot);
    std::string field = key.substr(lastDot + 1);

    // Try int -> bool -> string
    int32_t intValue = 0;
    if (settingMgr->GetIntValue(userId, settingKey, field, intValue)) {
        value = std::to_string(intValue);
        return true;
    }

    bool boolValue = false;
    if (settingMgr->GetBoolValue(userId, settingKey, field, boolValue)) {
        value = boolValue ? "true" : "false";
        return true;
    }

    std::string strValue;
    if (settingMgr->GetStringValue(userId, settingKey, field, strValue)) {
        value = strValue;
        return true;
    }

    MMI_HILOGW("Failed to get setting value for key: %{public}s", key.c_str());
    return false;
}

实现RegisterSettingObserver：
```cpp
sptr<SettingObserver> InputPlugin::RegisterSettingObserver(
    const std::string& uri,
    const std::string& key,
    SettingObserver::UpdateFunc callback)
{
    // Validate parameters
    if (uri.empty() || key.empty() || !callback) {
        MMI_HILOGE("Invalid parameters");
        return nullptr;
    }

    MMI_HILOGI("Plugin '%{public}s' registering observer: uri=%{public}s, key=%{public}s",
               name_.c_str(), uri.c_str(), key.c_str());

    // Create and register observer
    auto& settingHelper = SettingDataShare::GetInstance(MULTIMODAL_INPUT_SERVICE_ID);
    sptr<SettingObserver> observer = settingHelper.CreateObserver(key, callback);
    if (observer == nullptr) {
        MMI_HILOGE("Failed to create observer");
        return nullptr;
    }

    ErrCode ret = settingHelper.RegisterObserver(observer, uri);
    if (ret != ERR_OK) {
        MMI_HILOGE("Failed to register observer, error=%{public}d", ret);
        return nullptr;
    }

    // Track observer for lifecycle management
    {
        std::lock_guard<std::mutex> lock(observersMutex_);
        observers_.push_back(observer);
    }

    return observer;
}
```

bool InputPlugin::UnregisterSettingObserver(const std::string& uri,
                                            sptr<SettingObserver> observer)
{
    if (uri.empty() || observer == nullptr) {
        MMI_HILOGE("Invalid parameters");
        return false;
    }

    auto& settingHelper = SettingDataShare::GetInstance(MULTIMODAL_INPUT_SERVICE_ID);
    ErrCode ret = settingHelper.UnregisterObserver(observer, uri);

    if (ret != ERR_OK) {
        MMI_HILOGW("Failed to unregister observer, error=%{public}d", ret);
        return false;
    }

    // Remove from tracking list
    {
        std::lock_guard<std::mutex> lock(observersMutex_);
        auto it = std::find(observers_.begin(), observers_.end(), observer);
        if (it != observers_.end()) {
            observers_.erase(it);
        }
    }

    MMI_HILOGI("Plugin '%{public}s' unregistered observer successfully", name_.c_str());
    return true;
}

```cpp
        return false;

    if (ret != ERR_OK) {
        MMI_HILOGW("Failed to unregister observer, error=%{public}d", ret);
        return false;

        std::lock_guard<std::mutex> lock(observersMutex_);
        auto it = std::find(observers_.begin(), observers_.end(), observer);
        if (it != observers_.end()) {
```

```cpp
InputPlugin::~InputPlugin()
{
    // Cleanup all observers before destruction
    {
        std::lock_guard<std::mutex> lock(observersMutex_);
        for (auto& observer : observers_) {
            if (observer != nullptr) {
                UnregisterSettingObserver(observer);
            }
        }
        observers_.clear();
    }

    // Existing cleanup code
    if (handle_) {
        dlclose(handle_);
        handle_ = nullptr;
    }
    MMI_HILOGI("~InputPlugin");
}
```

### 5.3 监听配置变更（带DataShare就绪检查）

```cpp
class KeyboardPlugin : public IInputPlugin {
public:
    void Init(std::shared_ptr<IPluginContext> context) {
        context_ = context;

        // 检查DataShare是否就绪
        if (!context_->IsDataShareReady()) {
            MMI_HILOGW("DataShare not ready, using default configuration");
            currentRepeatRate_ = 50;  // 使用默认值
            return;
        }

        // 读取初始配置
        std::string valueStr;
        if (context_->GetSettingValue("settings.input.keyboard.repeatRate", valueStr)) {
            currentRepeatRate_ = std::stoi(valueStr);
        }

        // 监听配置变更
        auto callback = [this](const std::string& key) {
            std::string newValueStr;
            if (context_->GetSettingValue(key, newValueStr)) {
                try {
                    int newRate = std::stoi(newValueStr);
                    OnRepeatRateChanged(newRate);
                } catch (const std::exception& e) {
                    MMI_HILOGE("Failed to parse: %{public}s", e.what());
                }
            }
        };

        observer_ = context_->RegisterSettingObserver(
            "settings.input.keyboard.repeatRate",
            callback);
    }

    ~KeyboardPlugin() {
        if (observer_ != nullptr && context_ != nullptr) {
            context_->UnregisterSettingObserver(observer_);
        }
    }

private:
    std::shared_ptr<IPluginContext> context_;
    sptr<SettingObserver> observer_;
    int currentRepeatRate_ = 50;

    void OnRepeatRateChanged(int newRate) {
        currentRepeatRate_ = newRate;
        MMI_HILOGI("Repeat rate changed to: %{public}d", newRate);
    }
};
```

## 7. 技术要点

### 7.1 字符串统一返回的优势
- **通用性**：支持任意数据类型（int、bool、string、json等）
- **灵活性**：插件可以根据需求自行解析
- **简洁性**：框架接口简单，只有一个GetSettingValue方法
- **扩展性**：未来支持新类型无需修改框架

### 7.2 DataShare就绪状态检查的重要性
- **避免阻塞**：在DataShare未就绪时调用读取和监听接口可能导致阻塞
- **提高启动性能**：插件可以使用默认配置，避免等待DataShare就绪
- **增强稳定性**：防止因DataShare未就绪导致的超时和错误
- **灵活处理**：插件可以选择延迟加载或使用默认值

### 7.3 插件解析责任
- 插件需要知道配置的数据类型
- 插件负责类型转换和错误处理
- 插件可以使用任何解析库（json、xml等）

### 7.4 线程安全
- observers_列表使用mutex保护
- SettingManager内部已有线程安全机制
- 观察者回调在SettingDataShare的线程中执行

### 7.5 资源管理
- 析构函数自动清理所有观察者
- 使用智能指针（sptr）管理观察者生命周期
- 避免内存泄漏

### 7.6 错误处理
- GetSettingValue返回bool表示成功/失败
- 插件需要处理解析异常（try-catch）
- 失败时记录详细日志

### 7.7 性能优化
- GetSettingValue从SettingManager缓存读取
- 观察者注册是一次性开销
- 配置变更通知是异步的

## 8. 总结

### 8.1 接口设计
- **GetSettingValue(uri, key, value)**: 通用的key→value读取，返回字符串，URI作为参数
- **RegisterSettingObserver(uri, key, callback)**: 配置变更监听，URI作为参数
- **UnregisterSettingObserver(uri, observer)**: 注销监听，URI作为参数
- **IsDataShareReady()**: DataShare就绪状态查询

### 8.2 实现要点
- GetSettingValue从URI中提取userId，然后尝试int→bool→string的顺序读取，统一转换为字符串返回
- RegisterSettingObserver直接使用传入的URI注册观察者
- UnregisterSettingObserver使用传入的URI注销观察者
- IsDataShareReady委托给SettingManager的IsDatabaseReady()方法
- 析构函数自动清理所有观察者，防止泄漏

### 8.3 使用建议
- 插件在读取配置前应先调用IsDataShareReady()检查
- 插件需要自行构造URI（通过GetCurrentAccountId获取userId）
- 如果DataShare未就绪，插件可以选择使用默认配置或延迟加载
- 插件负责value的类型转换和异常处理
- 支持任意数据格式（int、bool、string、json等）
- URI作为参数提供了更大的灵活性，插件可以访问不同用户或不同数据源的配置
